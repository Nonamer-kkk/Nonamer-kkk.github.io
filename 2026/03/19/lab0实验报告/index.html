  • 执行add命令后，README.txt进入暂存区。
  • 再经过commit后，README.txt被加入版本库。
  • 最后被修改，README.txt处于已跟踪但修改未暂存状态。
  • 这与第一次执行add命令前的未跟踪状态不相同，因为未跟踪文件并未被Git管理，而此时文件已被跟踪，只是工作区文件与版本库不相同。

难点分析

lab0实验主要是使用命令行进行操作，这对之前几乎没使用过命令行的我来说是个挑战。我觉得本次实验的难点主要有三个方面。

第一就是脚本中一些复杂命令的使用
  • 因为sed,awk等等这些命令过于灵活，他们的使用方法也相当复杂，熟练掌握这些命令是本次实验的重点也是难点，指导书中的用法比较基础，给的例子也比较少，要想熟练使用的话，最好自己多查一查资料并练习。
第二就是shell脚本中的语法和各种结构

shell脚本的语法与之前习惯的语言很不同，常见的易错点有：

  • 赋值是不能有空格
  • []作为判断条件时必须有空格，也可以用[[]]
  • (())表示运算
  • 使用变量时要用$，而且在’’中的变量也要用’’括起来，或者直接使用””
第三就是Git的使用了
  • 虽然lab0对于Git的考察不多，但是能够熟练使用Git还是相当有用处的，怎么实现版本回退，如何撤销删除呀这些操作要是能够熟练掌握，将对以后大有裨益。重要的操作是：
    git checkout -- <file>：从版本库恢复文件到工作区。
    git reset HEAD <file>：撤销暂存区的修改。
    git rm --cached <file>：移除暂存区文件但保留工作区。

实验体会

lab0实验让我接触到了gcc,Makefile,git还有脚本这些工程中常用的操作，这是我最大的收获，还有就是通过Makefile的编写等加深了我对C语言和编译的理解。刚开始接触到这种命令行界面，虽然经常听人说这样会提高效率，但是我总是感觉不习惯，效率反而降低了，不过随着练习的加多，看着一行行命令也感觉越来越顺眼了，希望以后能够真正熟练使用命令行，提升自己的能力。
